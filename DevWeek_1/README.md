## Best Revision Order

Use this order when you want concepts to feel connected:

1. `0JS(SyntaxMastry)` - syntax warm-up
2. `JS1-(Var-Scopes-Closure&JsEssentials)` - memory, scope, closures, objects
3. `JS4-EventLoop` - execution flow behind async
4. `JS2-AsynchJs` - callbacks, promises, async/await
5. `JS3-Debounce&Throttle` - applied browser-style optimization
6. `JS5-Modules` - code organization and reuse
7. `JS6-RevisionSheet(Questions)` - interview recall round
8. `JS7-TypeScript-Awareness` - awareness only, not first-pass priority

Why `JS4-EventLoop` before `JS2-AsynchJs`:
Event loop understanding makes async behavior much easier to reason about.

## Fast Revision Method

For every section:

1. read the main note once
2. say the concept aloud in simple words
3. open the related practice files
4. predict output before reading full logic
5. close the file and rewrite key parts from memory later

Do not only read.
Always mix reading, speaking, and re-writing.
